## 第二课：监督学习 — 机器到底是怎么"学"的

### 手写规则 vs 机器学习

**传统编程（手写规则）：**
```python
if 颜色 == "红" and 大小 == "小" and 形状 == "圆":
    return "苹果"
elif 颜色 == "黄" and 大小 == "大" and 形状 == "椭圆":
    return "梨"
```
问题：遇到青苹果和小黄梨就崩了。

**机器学习：**
给 100 个标注好的水果（特征 + 标签），让机器自己找规律。

### 监督学习三要素

| 要素 | 是什么 | 水果摊例子 |
|------|--------|-----------|
| **特征（Features）** | 描述每个样本的数据 | 颜色、大小、形状 |
| **标签（Labels）** | 正确答案 | "苹果"或"梨" |
| **模型（Model）** | 学完后能预测的东西 | 店员脑子里的判断力 |

```
┌──────────┐      ┌──────────┐      ┌──────────┐
│  数据     │      │  算法     │      │  模型     │
│ (水果+标签)│ ───►│ (学习过程) │ ───►│ (学会的规律)│
└──────────┘      └──────────┘      └──────────┘
                                         │
                                   预测新水果
                                         │
                                         ▼
                                  ┌──────────┐
                                  │ [绿,中,圆]│
                                  │ → 苹果  ✓│
                                  └──────────┘
```

### 机器学习的训练过程（核心）

```
瞎猜 → 看误差 → 往误差变小的方向调整 → 再猜 → 直到误差接近零
```

三个核心概念：

| 概念 | 大白话 |
|------|--------|
| **损失函数 (Loss)** | 猜得有多错 |
| **梯度下降 (Gradient Descent)** | 往哪个方向调能让误差变小 |
| **参数 (Parameters)** | 机器在调的那个数 |

### 人脑 vs 机器

y = x² 的数据摆在面前，你看了 5 行就发现规律。机器做的事和你一样——只是你需要 5 行，机器可能需要几百行。**监督学习本质上就是"看数据、找规律"。**

### 现实世界的数据有噪声

真实数据从来不是完美的数学关系，而是"大致符合规律 + 随机误差"。

### 动手环节（进行中）

用 Python 生成带有噪声的线性数据，为下一课的"让机器自己学"做准备。

## 第四课：神经网络

### 核心思想

> 一堆简单的计算单元，一层层叠起来，就能表达非常复杂的规律。

### 一个神经元 = 画一条直线

```
输入 x₁, x₂, x₃ → 加权求和 → 过"门"（激活函数）→ 输出 0 或 1
```

单个神经元只能做线性分类（画直线分开两类东西）。

### 为什么需要深度？

一层只能画直线。多层可以画任意复杂的边界。

> **学生自己悟出（非常深刻）：** "类似傅里叶变换，用微小的确定性的规律叠加去逼近真实世界的实际。模拟信号数字化。"
> → 这就是**通用逼近定理**的通俗表达。

### 前向传播 + 反向传播

```
① Forward: 数据从输入流到输出，算出预测值
② Loss:    "预测"和"正确答案"差多少？
③ Backward: 从输出层往回，逐层分配误差责任
④ Update:   每个权重往误差变小的方向微调
⑤ 重复
```

**反向传播和 IC debug 的思路一模一样**——从 fail 点往回追溯，找到根因，只修那个点。

### 学生关键洞察

> "机器学习没有逻辑思维能力不会推理，它只会暴力的使用大量数据去逐渐逼近哪个正确的结果"
> → **正确。当前 AI 本质是超大规模模式匹配，不是有意识的思考者。**

### 输入层/输出层神经元数

| | 输入层 | 输出层 |
|---|---|---|
| 神经元数 | = 特征数 | = 要预测的目标数 |
| 例子 | 100 个覆盖率指标 → 100 个神经元 | 1 个 Bug 概率 → 1 个神经元 |

### 小测验回顾

1. 为什么一个神经元不够？→ 多维度的输入需要多神经元叠加（✅ 答对了本质）
2. 反向传播在干什么？→ 把误差信息往回传，迭代降低误差（✅ 方向对）
3. 100 特征 → 输出 1 个数，各层神经元数？→ 输入 100，输出 1（✅）

## 第五课：深度学习

> 把 3 层堆到 50 层、100 层、1000 层，不只是量的变化——是质的飞跃。

---

### 5.1 深度到底在干什么？—— 逐层抽象

用"认一张人脸照片"来感受：

```
输入层（像素）    →  一堆 RGB 数值（比如 224×224×3 = 15 万个数字）
第1层            →  边缘（横线、竖线、斜线）
第2层            →  简单形状（圆、方、角）—— 由边缘拼成
第3层            →  局部特征（眼睛、鼻子、嘴）—— 由形状拼成
第4层 ~ 第N层     →  完整人脸 —— "这是张人脸"
```

**浅层网络（3层）**：像素 → 边缘 → 形状 → 输出，跳过了"眼睛鼻子嘴"这个中间概念，直接硬猜，准确率上不去。

**深层网络（50层）**：每一步只做一点点抽象，层层递进，最后水到渠成。

> **关键洞察：深层 = 把一个大问题拆成很多个小步骤，每一步只解决一个小问题。**

**类比 IC 验证 debug：**

|  | 浅层做法 | 深层做法 |
|--|---------|---------|
| 思路 | 看到 fail → 直接猜根因 | fail → 定位模块 → 追踪信号 → 看波形时序 → 找到根因 |
| 效果 | 猜不准 | 一步步来，准 |

---

### 5.2 为什么以前不堆深？

不是不想堆，是堆不深——三个拦路虎：

| 问题 | 大白话 | 后果 |
|------|--------|------|
| **梯度消失** | 反向传播误差信号，每传一层就打一次折，传到前几层几乎为零 | 前面的层学不动 |
| **计算力不够** | 几十层 × 几百万数据，CPU 算不过来 | 等不起 |
| **数据不够** | 深层网络参数多，数据少就"背答案"（过拟合）| 实际用不了 |

---

### 5.3 三大突破，让"深"成为可能

#### 突破一：ReLU 激活函数 — 解决梯度消失

**先理解"梯度消失"是什么：**

反向传播 = 从输出层往回走，每过一层，误差信号要乘以一个数（导数）。

- Sigmoid 函数：导数的最大值才 0.25
- 传 10 层：0.25¹⁰ ≈ 0.00000095
- 信号几乎为 0 → 前面的层收不到"该怎么改"的信息 → 学不动

**ReLU 怎么解决：**

```
Sigmoid(x) = 1/(1+e^(-x))    → x很大或很小时，曲线是平的 → 导数≈0 → 梯度消失
ReLU(x)    = max(0, x)       → x>0 时，就是一条斜率为1的直线 → 导数=1 → 梯度不衰减
```

> **简单理解：Sigmoid 像传话游戏，每传一个人信息就模糊一点。ReLU 对"正信号"直接原话传过去，不损失。**

画出两个函数的形状：

```
Sigmoid:              ReLU:
  1 |     ----           |
    |   /                |        /
0.5 |  /                 |       /
    | /                  |      /
  0 |/                   |_____/
     +---------           +---------
     S 形，两端平          正数直接过，负数截断
```

#### 突破二：GPU 并行计算 — 解决算力

神经网络的计算本质：海量的矩阵乘法。

```
CPU：8 个大学生，聪明但人少，一个一个算
GPU：几千个小学生，不会复杂的但人多，同时算
矩阵乘法恰好是"小学生能干、但量特别大"的活 → GPU 完美适配
```

#### 突破三：大数据时代 — 解决数据

ImageNet（1400 万张标注图片）、互联网文本、传感器数据……数据不再稀缺。

---

### 5.4 两个"驯服技巧"——让 100 层网络乖乖训练

即使有了上面三者，直接堆 100 层还是会出问题。

#### Batch Normalization（批归一化，简称 BN）

**问题：** 各层之间在"互相拆台"。

前面层的权重一调整，后面层收到的数据分布就变了。后面层刚适应，前面层又变了——训练一直在震荡，收敛不了。

**方案：** 每一层输出之后，强行把数据"标准化"一下——均值归 0，方差归 1。

> **类比：** 炒菜时每个步骤之后都尝一口，保证味道在正轨上，不等到最后才发现糊了。

#### Residual Connection（残差连接 / 跳线）

**问题：** 52 层的网络，错误率反而比 20 层还高。不是过拟合——连训练集上都是 52 层更差。

**根本原因：** 层多了，信息在层间传递时不断被扭曲，越传越偏。

**方案（ResNet, 2015）：** 加"跳线"，隔几层就抄一条近道。

```
普通连接：
  输入 x → [Layer 1] → [Layer 2] → 输出 F(x)
  目标：学会从 x 变成目标 y 的完整映射

残差连接：
  输入 x → [Layer 1] → [Layer 2] → F(x) + x = 输出
              |                        ↑
              +-------- 跳线 ----------+
  
  目标：只学"残差"——即 y-x 的差。如果这几层没用，最差就是 F(x)=0，x 原样传给下一组。
```

> **精髓：** 不学"完整答案"，只学"还需要补什么"。最坏情况是"不补"（原样传递），而不会越改越糟。

**ResNet 的意义：** 有了跳线，网络从 20 层直接干到 152 层、甚至 1000+ 层，错误率持续下降。

### 5.7 小测验（附详细讲解）

#### 问题 1：为什么"深"比"宽"好？

讲之前先搞清楚什么是"宽"、什么是"深"：

```
宽（一层很多神经元）：
  输入 → [1000个神经元 挤在一层] → 输出
  
深（多层，每层适量）：
  输入 → [100个] → [100个] → [100个] → ... (10层) → 输出
```

为什么深更好？用**逐层抽象**来理解：

认人脸：
- 宽方案：一层里放 10000 个神经元，让它从像素直接判断"这是不是人脸"——太难了，要同时学边缘、形状、器官、整体。
- 深方案：第一层学边缘，第二层学形状，第三层学器官，第四层判断人脸——每层任务简单，越学越高级。

> **类比：**
> - 宽 = 一个人同时干切菜、炒菜、调味、摆盘，手忙脚乱
> - 深 = 流水线，每人只干一件事，效率高、质量好

#### 问题 2：梯度消失是什么？ReLU 怎么解决？

**梯度消失（大白话版）：**

训练神经网络 = 不断微调每个权重。

微调方向从哪来？从误差反向传播来。传播过程中，信号每过一层就要乘以一个小于 1 的数。

```
输出层误差 = 100%
传到倒数第1层 = 100% × 0.25 = 25%
传到倒数第2层 = 25% × 0.25 = 6.25%
传到倒数第3层 = 6.25% × 0.25 = 1.56%
...
传到第1层     ≈ 0.0000000001%  ← 信号消失了！
```

前面的层收不到信号 → 不知道怎么调 → 不动 → 等于白搭。

**ReLU 怎么解决：**

```
Sigmoid 导数（决定信号打多少折）：
  最大 0.25，两端趋近 0

ReLU 导数：
  x > 0 时，导数 = 1（不打折！）
  x ≤ 0 时，导数 = 0（这个神经元这次不参与）
```

> **所以：ReLU 让正信号"原价"传递，不打折，深层也能收到信号。**

#### 问题 3：残差连接为什么加跳线？

**先说问题：**

直觉上：层越多 → 能力越强 → 效果越好。

实际上（ResNet 之前）：52 层反而比 20 层效果差。不是过拟合（训练集上就差），就是训不动。

**原因：**

每一层都可能引入一点"误差"（信息被微小的扭曲）。层数越多，扭曲累积越多。到后面，信息已经完全变形了。

**解决方案：**

加跳线。信息有两条路可以走：
- 主路：经过 Layer 1 → Layer 2（学"需要改什么"）
- 短路：直接跳过去（原样保留）

两条路的输出加在一起。如果主路学歪了，短路保证了原始信息不丢失。

> **类比：** 改代码时用 git。残差连接就像 git commit——你的改动（残差）是增量，原始代码（输入 x）始终保留在历史里。出了问题可以回退，至少不会比原来更差。

#### 问题 4：你工作中有什么可以用"逐层抽象"来理解？

这个问题没有标准答案，是让你把"深层网络"的思路用到自己的工作上。举几个可能的例子：

- **Debug 流程**：fail 信息（输入）→ 定位模块 → 追踪可疑信号 → 看具体波形 → 确定根因（输出），每一层都在上一层的基础上缩小范围、提升精度。
- **读 Spec 文档**：全文 → 找到与我相关的章节 → 提取验证要点 → 列出测试场景 → 写测试用例。
- **学一个新工具**：知道它能干什么 → 会跑 demo → 理解原理 → 能改参数 → 能自己写脚本。

### 5.8 课堂 Q&A

#### Q1（学生总结）：深度学习的本质理解

> 深度学习实际上是用多层次对应多个步骤，每个步骤解决一个"问题"，每个问题对应多个特征。这些问题相当于是对总问题的特征的分类，按照分类一步一步解决问题，这样降低了每个层次的难度。并且解决了需要人工输入特征的不便。但是引入了反向传递时梯度下降的问题，也就是最终结果反馈回各层，让各层调整自己的参数时，有效信息会减少。

**回复（Claude）：** 理解完全正确。补充一点：梯度消失不是"深度本身"的错，而是 Sigmoid 激活函数 + 深度共同导致的。换了 ReLU 之后，梯度消失大幅缓解。所以现在几百层的网络也能训。

> 深层网络 = 把一个复杂问题自动拆成若干层，每层学不同抽象级别的特征，层层递进。代价是反向传播时信息可能衰减（梯度消失），但 ReLU 和残差连接基本解决了这个问题。

---

#### Q2（学生追问）：网络怎么知道要提取什么特征？

> 为什么多层能够自己提取特征？神经网络是根据什么提取出特征的？怎么提取出来的就是人们想要的特征呢？比如第一层网络提取边缘，它怎么知道要提取和边缘有关的特征呢？

**回复（Claude）：**

**核心答案：它不知道。** 权重一开始是随机的，没人告诉任何一层"你去学边缘"。

边缘检测能力是**涌现（emerge）**出来的——不是被人设计的，是被数据和目标函数逼出来的。

**整个过程：**

1. 随机初始化 → 瞎猜
2. 算误差 → 反向传播 → 微调权重
3. 重复几百万次

**为什么第一层最终学会了边缘而不是别的东西？**

因为"检测边缘"对降低误差最有效。第一层每个神经元只有很小的感受野（比如 3×3 像素），在自然图像中，相邻像素强相关，物体边界处像素值剧烈变化——边缘是信息最密集的地方。神经网络发现：先找边缘 → 信息保留最多 → 往后的层更容易判断 → 误差降得最快。

> **不是"想"出来的，是被数据"逼"出来的。**

**类比 IC 布局布线：** EDA 工具不是被人告诉"这根线走这里"，而是给了网表 + 约束 + 目标函数，工具自己通过优化找到最优方案。神经网络也一样——给了数据 + 标签 + 损失函数，最佳特征自动涌现。

**一个惊人的事实：** 几乎所有在自然图像上训练的神经网络，第一层都会自动学会类似 Gabor 滤波器的边缘检测器。而人类视觉皮层（V1 区）的神经元也在做一模一样的事——神经科学和深度学习独立发现了相同的"最优方案"。这说明边缘检测不是设计选择，而是自然图像本身的结构决定的。

**每一层都没有被告诉该学什么。每一层只是在上一层的输出基础上，继续做"怎样让误差最小"这个唯一的任务。**

> **总结：神经网络的特征是涌现出来的——不是因为有人设计，而是因为数据 + 目标函数的双重压力，通过梯度下降，逼出了最有效的特征表示。**

## 第六课：CNN（卷积神经网络）— 让机器"看"懂图像

### 6.0 课前 Q&A：卷积核 vs 神经元

> **学生 Q1：** 全连接同时存在 1000 个神经元，神经元的输出等于 15 万个像素点分别乘以自身的权重累加和。卷积核的运算不是这样，而是以 3×3 为单位对整个图片遍历进行卷积，这样每个神经元只维护 9 个权重参数，然后输出的不是一个数值而是一个特征图也就是一个矩阵数据。是这样吗？
>
> **学生 Q2：** 如果是这样，这个神经元在一次卷积的过程中这 9 个权重值变化吗？是否不变，等下一次反向传播时才变？
>
> **学生 Q3：** 全连接方式相当于每个神经元一次性输入 15 万个像素点，一次性计算出结果。而 CNN 方式每个神经元实际上进行的操作是卷积核扫描完整张图的最后结果。32 个卷积核同时扫描，都扫描完之后出 32 个结果，相当于是 32 个神经元。神经元概念和卷积核概念的关系就是卷积核通过多次扫描分步运算完成之前所谓一个神经元的工作。

**回复（Claude）：**

Q1 — 完全正确：

| | 全连接层 | 卷积层 |
|---|---|---|
| 一个单元处理多少像素 | 全部 15 万个 | 只有 3×3 = 9 个 |
| 权重数 | 每个神经元 15 万个独立权重 | 整个卷积核只有 9 个权重，全图共享 |
| 输出 | 1 个数（标量） | 一整张特征图（矩阵） |
| 1000 个单元 | 1000 个神经元，各出 1 个数 | 比如 32 个卷积核 → 32 张特征图 |

Q2 — 对，正向传播时 9 个权重**不变**。扫完整张图后才反向传播更新一次。和全连接网络完全一样，唯一区别就是**同一套权重被整张图的所有位置共享**。

Q3 — 方向完全对，但需要微调定义：

> 卷积核 ≠ 一个神经元。卷积核 = 一个**可复用的特征探测器**。它在图的每个位置相当于一个只看局部的神经元（9→1），同一套参数在所有位置共享。

```
全连接层：                          卷积层：

神经元1 ─ 15万输入 → 输出1个数      卷积核1 (9个权重)，扫完整图 → 特征图1 (矩阵)
神经元2 ─ 15万输入 → 输出1个数      卷积核2 (9个权重)，扫完整图 → 特征图2 (矩阵)
神经元3 ─ 15万输入 → 输出1个数      卷积核3 (9个权重)，扫完整图 → 特征图3 (矩阵)
...                                 ...
每个神经元：15万个权重，独立          每个卷积核：9个权重，全图共享
输出：都是标量                       输出：都是特征图（矩阵）
```

### 6.1 为什么需要 CNN？

一张 224×224 的彩色图片，如果全连接：

```
输入：224 × 224 × 3 = 150,528 个像素值
第一层 1000 个神经元 → 150,528 × 1000 = 1.5 亿个权重参数
```

1.5 亿参数，而且**完全忽略了像素之间的空间关系**——相邻像素有关联，全连接当成了孤立的数。

CNN 解决两个问题：**参数爆炸** + **空间结构被忽略**。

---

### 6.2 卷积核 = 一个能滑动的"特征探测器"

卷积核是一个小矩阵（如 3×3），在图片上从左到右、从上到下滑动，每次只"看"一小块。

```
图片（10×10 像素）：
┌─────────────────┐
│                 │
│   ┌───┐         │  ← 3×3 卷积核，只看这一小块
│   │   │         │
│   └───┘         │
│                 │
└─────────────────┘

卷积核从左到右、从上到下滑动，每滑一步算一个值，最后得到一张"特征图"
```

**一个卷积核 = 一个模式探测器。** 比如一个卷积核最终学会了检测"竖边"——遇到竖边的地方输出高值。

```
图片：             卷积核（竖边探测器）：     输出的特征图：
┌──────────┐      ┌──────────┐         ┌──────────┐
│  ██░░██   │      │ -1  0  1 │         │   ░░██░░  │  ← 亮处 = 检测到竖边
│  ██░░██   │  ✕   │ -1  0  1 │    =    │   ░░██░░  │
│  ██░░██   │      │ -1  0  1 │         │   ░░██░░  │
│  ██░░██   │      └──────────┘         └──────────┘
└──────────┘
  左黑右白 → 天然竖边
```

> 卷积操作 = 把卷积核盖在图片上，对应位置相乘再求和。得分越高 → 这块区域和卷积核的模式越匹配。

---

### 6.3 四个核心概念

| 概念 | 是什么 | 大白话 |
|------|--------|--------|
| **卷积核（Kernel）** | 3×3 或 5×5 的小矩阵 | 一个模式探测器，比如"找竖边" |
| **特征图（Feature Map）** | 卷积核扫完整张图的结果 | 标记了"哪里有竖边"的地图 |
| **步长（Stride）** | 每次滑多远 | 步长=1 → 密集扫描；步长=2 → 跳着扫 |
| **填充（Padding）** | 图片外围补 0 | 保证边缘区域也能被扫描到，且输出尺寸可控 |

### 6.4 CNN 两大核心思想

#### 思想一：局部感受野

全连接：每个神经元看整张图 → 1.5 亿参数。
CNN：每个神经元只看 3×3 = 9 个像素 → 参数 = 卷积核大小，与图片尺寸无关。

```
全连接：每个神经元连全部像素     CNN：每个神经元只看一小块
  ┌──────────┐                     ┌──────────┐
  │ 全部 15万 │ ← 神经元1           │    ┌┐    │ ← 神经元只看这里
  │  像素    │ ← 神经元2           │    └┘    │
  │          │ ← 神经元3           │          │
  └──────────┘                     └──────────┘
```

#### 思想二：权重共享

普通网络：每个位置都学一套独立的权重。
CNN：**同一个卷积核在整张图的所有位置共享权重。** 不管"竖边"出现在左上角还是右下角，同一个卷积核都能检测到。

```
同一个竖边探测器 → 滑到任何位置都能工作
→ 参数从 1.5 亿 降到 几百
→ 学到的模式是"位置无关"的
```

### 6.5 Pooling（池化）— 压缩但不丢关键信息

**为什么需要压缩？** 特征图还是太大。32 个 224×224 特征图 = 约 160 万个值。一直这么大 → 计算量大 → 容易过拟合。

**池化 = 给特征图做"缩略图"。** 最常用的是**最大池化**（Max Pooling）：

```
原始特征图 4×4              2×2 窗口，取每个窗口最大值
┌─────────────┐            ┌─────┬─────┐
│ 1   3  │ 2   1│            │     │     │
│ 2  [8] │ 4   5│     →      │  8  │  5  │
│────────┼──────│            ├─────┼─────┤
│ 3   1  │ 7   2│            │     │     │
│ 4   2  │ 1  [8]│            │  4  │  8  │
└─────────────┘            └─────┴─────┘

取每个 2×2 窗口的最大值
```

**为什么取最大而不是平均？**

| 方式 | 含义 | 问题 |
|------|------|------|
| 取最大（Max Pooling） | "这里有没有竖边？"→ 有，保留 | 丢掉了"竖边在窗口里具体在哪" |
| 取平均（Avg Pooling） | "这里竖边多强？"→ 取平均强度 | 强信号被弱信号稀释了 |

取最大 = 只关心"有没有"，不关心"精确在哪"。判断图片里有猫，不需要知道猫耳朵在 (127, 89)，只需要知道"有耳朵特征"。

**Pooling 三个好处：**

1. 数据量暴降：2×2 池化 → 数据量 ÷4
2. 微小位移不敏感：猫往左挪 1 像素，池化后特征图基本不变
3. 防止过拟合：参数少了，不容易"背答案"

### 6.6 经典 CNN 结构

```
输入图片 (224×224×3)
     │
     ▼
Conv1: 32 个 3×3 卷积核 → 32 张特征图 (224×224)
     │  学：边缘、颜色
     ▼
ReLU: 负数归零
     │
     ▼
Pool1: 2×2 最大池化 → 32 张特征图 (112×112)    数据量 ÷4
     │
     ▼
Conv2: 64 个 3×3 卷积核 → 64 张特征图 (112×112)
     │  学：纹理、简单形状
     ▼
ReLU + Pool2 → 64 张特征图 (56×56)              数据量再 ÷4
     │
     ▼
Conv3: 128 个卷积核
     │  学：物体部件（眼睛、轮子）
     ▼
  ... 继续堆 ...
     │
     ▼
Flatten: 把所有特征图拉成一长条向量
     │
     ▼
全连接层 → 分类输出："猫 (95%)"
```

| 阶段 | 操作 | 干什么 |
|------|------|--------|
| 特征提取 | Conv + ReLU + Pool，反复几轮 | 从边缘→形状→局部部件，逐步抽象 |
| 分类 | Flatten → 全连接 → 输出 | 把所有特征摊平，综合判断"是什么" |

---

### 6.7 CNN 完美体现"逐层抽象"

| 层 | CNN 学到的 | 对应第五课 |
|----|-----------|-----------|

| Conv2 | 纹理、简单形状 | 中级特征 |
| Conv3 | 物体部件（眼睛、轮子） | 高级特征 |
| Conv4+ | 完整物体概念 | 语义级特征 |

## 第七课：Transformer — 现代 AI 的基石

> 2017 年，Google 发了一篇论文《Attention Is All You Need》，标题就在挑衅——**只要有注意力就够了，RNN 可以扔了。**

---

### 7.1 为什么要革 RNN 的命？

在 Transformer 之前，处理文字用的是 **RNN / LSTM**（循环网络）——一个词一个词地读：

```
"我 爱 吃 苹果"
 ↓  ↓  ↓  ↓
①→②→③→④    ← 必须按顺序，第 3 个词等前两个读完才能处理
```

三个致命问题：

| 问题 | 后果 |
|------|------|
| **不能并行** | 一个接一个，GPU 几千个核闲着 |
| **长距离忘事** | 读到第 100 个词，第 1 个词的信息丢光了 |
| **梯度消失** | 传 100 步信号衰减没（同第五课） |

---

### 7.2 核心思路：从"排队"变成"圆桌会议"

```
RNN（接力赛）：              Transformer（圆桌会议）：
我 → 爱 → 吃 → 苹果         我 ←──────→ 爱
  （必须排队）                │           │
                             │    大家    │
                             │   同时对话  │
                             ↓           ↓
                            吃 ←──────→ 苹果
                            
每个人都能直接跟任何其他人说话，不需要排队传话
```

---

### 7.3 自注意力（Self-Attention）— 核心引擎

**大白话：每个词问所有其他词——"你跟我有多相关？"**

例句：**"猫吃鱼因为它饿了"**——"它"指谁？

```
     "它" 问每一个词：→  "你跟我有多相关？"

     猫 ────────────→  相关度 0.9（高！"它"大概率指"猫"）
     吃 ────────────→  相关度 0.1（低）
     鱼 ────────────→  相关度 0.3（一般）
     因为 ──────────→  相关度 0.05（低）
     饿 ────────────→  相关度 0.2（低）
     了 ────────────→  相关度 0.0（几乎无关）
```

> "它"发现"猫"得分最高 → 知道自己大概率在指"猫"。

**每个词都做一遍：** "猫"发现自己跟"吃""饿"高度相关；"吃"发现自己跟"鱼"高度相关。

---

### 7.4 计算过程（理解逻辑即可，不要求记公式）

```
第一步：给每个词造三个向量 —— Q、K、V

  Q（Query，查询）：  "我想找什么？" — 这个词在问什么
  K（Key，键）：      "我代表什么？" — 这个词的标签
  V（Value，值）：    "我的实际信息是啥？" — 这个词的内容

第二步：用 Q 去匹配所有 K，算出注意力分数

  "它"的 Q  ✕  "猫"的 K = 0.9  ← 高匹配
  "它"的 Q  ✕  "吃"的 K = 0.1
  "它"的 Q  ✕  "鱼"的 K = 0.3
  ...

第三步：用分数当权重，对所有 V 加权求和

  最终"它"的输出 = 0.9×猫的内容 + 0.1×吃的内容 + 0.3×鱼的内容 + ...
                ≈ 主要包含了"猫"的信息
```

> **效果："它"的输出向量里融入了"猫"的信息。模型就知道"它 = 猫"。**

---

### 7.5 IC 类比：全芯片时序分析

```
Self-Attention ≈ 全芯片时序分析（Setup Time 检查）

每个寄存器（= 一个词）都要检查：
  → 发给我的数据路径上经过了哪些组合逻辑？
  → 哪些逻辑延时最长（得分最高）？
  → 把关键路径的信息汇总过来

寄存器不是只看邻居，而是"关注"整条路径上的所有门。
哪个门对时序贡献最大，注意力就放在那个门上。

RNN = 只能顺着路径一个门一个门看
Transformer = 所有寄存器同时看所有门，一次性算完
```

---

### 7.6 多头注意力（Multi-Head Attention）

一套 QKV = 一个"观察角度"。多个头 = 多个角度同时看：

```
头1（语法角度）：  "它"关注"猫"（代词→先行词）
头2（语义角度）：  "猫"关注"吃""鱼"（主语→动作→宾语）
头3（因果角度）：  "饿"关注"吃"（原因→行为）
...
```

> 和 CNN 多个卷积核道理一样——不同核检测不同模式，不同头关注不同关系。

---

### 7.7 位置编码（Positional Encoding）

Transformer 一次性看完所有词，那顺序信息怎么办？

"猫吃鱼" 和 "鱼吃猫" 意思完全不同。

**方案：给每个词的位置编一个独特信号，直接加到词向量里。**

```
词向量：    [0.2, 0.5, 0.1, ...]    ← 这个词"是什么"
+ 位置编码： [0.0, 0.8, 0.2, ...]    ← 这个词"在第几位"
─────────────────────────────────
= 最终输入： [0.2, 1.3, 0.3, ...]    ← 既知道"是什么"，也知道"在哪"
```

> 类似芯片设计里给每个信号加时间戳——信号值和时序信息一起编码，后面的分析就不会搞错顺序。

---

### 7.8 Transformer 整体结构

```
输入："我 爱 吃 苹果"
  │
  ├── 词嵌入（Embedding）：每个词 → 一个向量
  ├── + 位置编码
  │
  ▼
┌──────────────────────────────────┐
│          Encoder（编码器）         │
│   × N 层（原论文 N=6）             │
│                                  │
│   每层 = Self-Attention           │
│       + 前馈网络（类似全连接层）     │
│       + 残差连接 + BN             │  ← 第五课的残差和BN！
│                                  │
│   输出：每个词都被"上下文化"了       │
│         "苹果"不再是孤立词，而是融合了 │
│         "我""爱""吃"的信息          │
└──────────────────────────────────┘
  │
  ▼
┌──────────────────────────────────┐
│          Decoder（解码器）         │
│   × N 层                          │
│   逐词生成输出："我 → 爱 → 吃 → 苹果" │
└──────────────────────────────────┘
```

**Encoder 和 Decoder 的区别：**

- **Encoder**：读懂输入，把所有词互相"看一遍"，输出理解后的表示
- **Decoder**：根据 Encoder 的理解，一个字一个字地生成输出

---

### 7.9 为什么 Transformer 是革命性的？

| 对比 | RNN/LSTM | Transformer |
|------|----------|-------------|
| 处理方式 | 一个接一个（串行） | 全部一起看（并行） |
| 长距离依赖 | 梯度消失，记不住 | Self-Attention 直接连接任意距离 |
| 训练速度 | 慢（不能并行） | 快（GPU 满载） |
| 可扩展性 | 差 | 极强——万亿参数也能训 |

> 没有 Transformer，就没有 GPT、Claude、文心一言等所有现代大模型。

---

### 7.10 Transformer 家族三兄弟

```
Transformer（2017，原论文：Encoder + Decoder）
│
├── BERT 系列（Encoder-Only）
│   只保留 Encoder，理解型任务
│   情感分析、搜索、分类
│   → 读完给一个结论
│
├── GPT 系列（Decoder-Only）
│   只保留 Decoder，生成型任务
│   自回归：一个字接一个字地输出
│   → ChatGPT / Claude 都是这种
│
└── ViT（Vision Transformer）
    把图片切成小块（patch），每块当"一个词"
    → CNN 的挑战者
```

---

### 7.11 学过的所有内容 — 知识地图

```
AI
└── ML
    └── 神经网络 (NN)
        │ 核心：前向传播 + 反向传播 + 梯度下降
        │
        └── 深度学习 (DL)
            │ 核心：逐层抽象（低级→中级→高级特征）
            │ 关键技巧：ReLU、BatchNorm、残差连接
            │
            ├── CNN（图像空间）
            │   核心：局部感受野 + 权重共享
            │
            ├── RNN/LSTM（序列，已被取代）
            │
            └── Transformer（现在统治一切）
                核心：自注意力（Self-Attention）
                代表：GPT / Claude / BERT
```

## 第八课：大语言模型（LLM）— 从图纸到成品

> 把 Transformer 架构 + 海量数据 + 人类反馈，变成能对话的 AI。

---

### 8.1 Transformer vs LLM：建筑图纸 vs 房子

| 概念 | 是什么 | 类比 |
|------|--------|------|
| **Transformer** | 一种神经网络架构（2017）| 建筑图纸 |
| **GPT** | OpenAI 用 Transformer 训练的**具体模型** | 一栋具体的楼 |
| **Claude** | Anthropic 用 Transformer 训练的**另一个模型** | 另一栋楼 |
| **LLM** | 所有这些大模型的统称 | "高层建筑"这个类别 |

> Transformer = 技术方案。LLM = 用这个方案做出来的产品。
> GPT、Claude、文心一言、通义千问都是 LLM，都基于 Transformer。

---

### 8.2 对话全流程：从你输入到它回复

```
你："Python 怎么写 for 循环？"

  ① 把整个对话历史拼接成一条长文本：
     "<用户>：Python 怎么写 for 循环？
      <助手>："
     
  ② 分词器切成 token，塞进 Transformer
  
  ③ Transformer 处理，最后一层隐藏状态包含了对这句话的"理解"
  
  ④ 隐藏状态 → 输出层 → 词汇表概率分布
     for:   0.42  ← 采样选这个
     关:    0.01
     你:    0.03
     ...
     
  ⑤ "for"拼回输入 → 再来一轮：
     "...<助手>：for"
     → 输出：循环的写法
     
  ⑥ 循环，直到输出 <结束> 标记
  
最终输出："for i in range(10): print(i)"
```

> **模型不是"想好再说"——它是一边想一边说，每写一个字都把之前所有内容重新看一遍。**

---

### 8.3 训练的三个阶段

#### 阶段一：预训练（Pre-training）— 学语言的基本规律

```
找一段互联网文本，遮掉最后一个字，让模型猜：

原文："Python 的 for 循环语法简洁。"
遮掉："Python 的 for 循环语法简洁。[猜]"

模型猜："洁" ← 原文本就是这个字
猜对 ✓ → 奖励（更新权重）
猜错 ✗ → 惩罚（反向更新）

× 几万亿次
→ 模型学会了"像互联网文本那样说话"
```

**这个阶段不区分"问题"还是"聊天"——就是学文字的统计规律。**

预训练完的模型叫 **Base Model**。它只会"续写"——给它半句话，它往下接。但不知道你在"提问"。

#### 阶段二：指令微调（Instruction Tuning）— 学会"回答"

```
给模型看几万条"问题 → 答案"对：

输入：  "用户：Python 怎么写 for 循环？\n助手："
目标：  "你可以用 for item in iterable: 的语法..."

模型通过大量这种样本学到：
  → "用户："后面是问题
  → "助手："后面应该输出有用的回答
  → 这是在回答问题，不是在续写
```

#### 阶段三：RLHF（人类反馈强化学习）— 学会"什么是好回答"

同一个问题可以有很多答案，但有好坏之分：

```
问题："Python 怎么写 for 循环？"

回答 A："for i in range(10): print(i)  # 简洁直接"
回答 B："这个问题很好，让我来教你...（啰嗦了几百字）"
回答 C："用 while 循环（虽然也能实现但不对）"

人类标注者排序：A > B > C
       ↓
训练一个"奖励模型"学会给回答打分
       ↓
用奖励模型引导 LLM 倾向于生成 A 那样的好回答
```

| 训练阶段 | 教什么 | 数据量 | 谁判断好坏 |
|----------|--------|--------|------------|
| 预训练 | 语言统计规律 | 几万亿 token | 互联网文本 |
| 指令微调 | 问答格式 | 几万~几十万条 | 人类写的问答对 |
| RLHF | 什么是"好"回答 | 几万条偏好 | 人类标注者排序 |

---

### 8.4 同一句话，不同上下文 → 不同理解

这恰恰是 Self-Attention 发挥威力之处：

```
场景一：对话前面在说"取钱""账户"
  → "我去银行办点事"
  → Self-Attention 让"银行"和"取钱""账户"高关联
  → 理解：银行 = 金融机构

场景二：对话前面在说"河边散步"
  → "我去银行办点事"  
  → Self-Attention 让"银行"和"河""岸边"高关联
  → 理解：银行 = 河岸
```

> 同一个句子，周围上下文不同 → Self-Attention 分数不同 → 隐藏状态不同 → 理解不同。

---

### 8.5 语气怎么处理？

中文的语气信息来自语气词和标点，而分词器把它们也当成独立的 token：

```
"你来了"   → ["你", "来了"]
"你来了？" → ["你", "来了", "？"]
"你来了！" → ["你", "来了", "！"]
"你来了呢" → ["你", "来了", "呢"]
```

对模型来说，**"？""！""呢"就是不同的 token**，和不同的词一样对待。它们在训练数据里反复出现的上下文，教会了模型各自代表什么语气。

```
"等了三个小时，你来了" → 平静/抱怨
"终于！你来了！"       → 兴奋期待

不同的上下文 → Self-Attention 将"你来了"和不同的语气 token 关联
→ 不同的理解 → 不同的回复
```

---

### 8.6 聊天 vs 问答：对模型有区别吗？

**对模型来说没有区别——所有输入都是同样的文字拼接。**

但它在训练数据的海洋里学会了区分：

| 输入模式 | 训练中见过的类似文本 | 学会的模式 |
|----------|-------------------|-----------|
| "今天天气真好" | 论坛闲聊、微信对话 | 闲聊回复 |
| "Python 怎么写 for？" | 技术问答、教程 | 教学回复 |
| "帮我写一个函数" | 编程协作对话 | 协作模式 |

> 就像人凭经验知道"别人说天气是在寒暄"、"别人问语法是在请教"——模型也是凭训练经验。

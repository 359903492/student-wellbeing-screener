## 一、项目概述

一个轻量级的学生情绪/压力筛查 Web 工具。教师发布二维码，学生手机扫码填写 PHQ-8 + GAD-7 标准化量表（共15题），教师通过仪表盘查看全年级统计、需关注名单，支持 Excel 导出和数据一键备份。

**核心原则**：
- **非临床诊断工具**，仅用于筛查和分流
- 学生匿名填写（仅学号），保护隐私
- 零服务器成本，全部使用免费云服务

## 五、数据流

### 学生提交流程

```
1. 学生输入学号 → 点击"开始填写"
2. 检查学号是否重复（去前导零正规化后比较：001=0001=1）
   ├─ 重复 → 弹窗："该学号已提交过，确定要再次提交吗？"
   └─ 不重复 → 直接进入问卷
3. 逐一作答 15 题（点击选项自动跳转下一题）
4. 全部答完 → 点击"提交"
5. 显示"✅ 提交成功"页面（不显示分数）
6. 后台：GET /api → 追加新记录 → PUT /api 写回
```

### 教师查看流程

```
1. 打开 dashboard.html → 输入密码（默认 20260625）
2. 密码验证：优先 localStorage，其次云端同步，最后默认值
3. GET /api?_=timestamp → 获取全部提交记录（带缓存破坏参数）
4. 客户端计算：总人数、四级分布、需关注名单
5. 渲染 Chart.js 图表 + 学生表格 + 提交时间列
6. 每 5 分钟自动刷新
```

### 数据存储格式（jsonblob）

```json
{
  "password": "20260625",
  "submissions": [
    {
      "timestamp": "2025-06-25T10:30:00.000Z",
      "studentId": "20240101",
      "phq9_total": 8,
      "gad7_total": 6
    }
  ]
}
```

## 八、部署指南

### 部署命令

```bash
# 步骤 1：打包文件
cd 项目目录
python -c "
import zipfile, os
with zipfile.ZipFile('deploy.zip', 'w') as z:
    for f in ['student.html','dashboard.html','index.html','teacher.html','_redirects']:
        z.write(f)
print('OK')
"

# 步骤 2：上传到 Netlify
curl -X POST "https://api.netlify.com/api/v1/sites/173e0d71-16f7-4d8e-bf73-9fe8f112ddcc/deploys" \
  -H "Authorization: Bearer <NETLIFY_TOKEN>" \
  -H "Content-Type: application/zip" \
  --data-binary @deploy.zip

# 步骤 3：推送到 GitHub
git add -A && git commit -m "update" && git push github master
```

### 手动部署（Netlify 网页）

1. 浏览器打开 https://app.netlify.com/sites/student-wellbeing-site/deploys
2. 将 deploy.zip 拖拽到页面中
3. 等待 "Published" 状态

## 九、维护指南

### 日常操作

| 操作 | 频率 | 方法 |
|------|------|------|
| 查看数据 | 随时 | 打开仪表盘 |
| 备份数据 | 每次使用后 | 点击「💾 备份数据」+「导出 Excel」 |
| 保活数据 | 每 2-3 周 | 打开仪表盘访问一次 |
| 清空数据 | 换班级前 | 先备份 → 点「🗑 清空数据」 |
| 修改密码 | 需要时 | 点「⚙ 修改密码」 |

### ⚠️ 数据保活

jsonblob 是免费服务，**超过 30 天无人访问可能自动清理所有数据**。

- **保活方法**：每 2-3 周打开仪表盘一次，刷新即可
- **寒暑假前**：务必点击「💾 备份数据」+「导出 Excel」双重备份
- **恢复方法**：如果 jsonblob 数据被清理，需重新创建端点（POST jsonblob API）并导入备份

### 修改代码流程

1. 修改本地 HTML 文件
2. 打包 zip → 运行部署命令 → 等 10 秒生效
3. git commit + push 到 GitHub（保存历史）

### 关键接口约定

- `GET /api` → 返回 `{ password, submissions: [...] }`
- `PUT /api` + body `{ password, submissions }` → 更新数据
- `_redirects` 将 `/api` 代理到 jsonblob（Netlify 特有功能，GitHub Pages 不支持）
- 所有前端 GET 请求必须带 `?_=timestamp` 参数防止缓存
